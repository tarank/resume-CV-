Position Scheme

This website use a combination of positioning to better fit the content.

majority use same static positioning.

Breif explanation of website:

relative positioning
  • the navigation menu, a child of nav, is positioned lower (relative to its original place) to ensure it is not covered by the navigation icon. (navigation.css, Line 17)
  • the form in home page is set to relative to serve as positioning ancestor for the input and select. (see the next item, homepage.css, Line 81)
absolute positioning
the form input and select in home page is positioned with 10% right empty space (relative to the form) to be aligned vertically. This also takes the elements out of normal flow. (homepage.css
fixed positioning
the nav class and the navigation icon is fixed to the top right corner of the browser body. This also takes the elements out of normal flow. (navigation.css

CSS3 Features

Many CSS3 features are introduced in this website, including but not limited to:

Round Corner
  • border-radius is used to create round corner for many div elements. (default.css, Line 6, 10)
  • border-radius is also used for menu_label in order page, with only topleft and topright corner rounded. (order.css, Line 36)
Box Shadow
box-shadow is used to decorate the form in the homepage. (homepage.css, Line 82)
Transitions
  • transition is used to create a color and background color change effect when the mouse hovers over an item in the navigation menu. (navigation.css
  • A similar effect (background color change) is applied for the menu_label in order page. (order.css
Animations
  • When the page is loaded, animation makes the navigation icon flys in with rotation, and then rotate for another 360 degrees. (default.css
  • At the same time, the navigation menu fades in. (default.css

Style grouping

All the CSS content is grouped into 6 files in the /css folder, and HTML files are linked to them.

No inline or embedded style is used in this website.

The files are:

default.css
the common theme across the site, for index.html and order.html (on screen).
Including the header and welcome footer and border style
navigation.css
navigation style, for all pages (on screen, that is, will be hidden in print).
Including the navigation icon navigation menu and keyframe functions
homepage.css
specific style for index.html.
Including the two-column layout , information block , promotion blockbooking block
order.css
specific style for order.html (on screen).
Including the two-column layout , order menu and ordered item
printOrder.css
specific style for printing the order.html.
All the blocks other than the header and ordered item are not displayed.
These two blocks are also adjusted to fit the print width. Header , ordered item .
design.css
specific style for this page, design.html.
